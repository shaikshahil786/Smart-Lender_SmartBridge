# Smart Lender

ML-powered loan eligibility screening. Trains four classifiers (Decision
Tree, Random Forest, KNN, XGBoost) on historical loan applicant data,
automatically keeps whichever one tests best, and serves it through a Flask
web app for real-time predictions.

```
ApplicantIncome, CoapplicantIncome, LoanAmount, Loan_Amount_Term,
Credit_History, Gender, Married, Dependents, Education,
Self_Employed, Property_Area   --->   Loan_Status (Y / N)
```

## Setup

```bash
cd smart_lender
python3 -m venv venv
source venv/bin/activate          # venv\Scripts\activate on Windows
pip install -r requirements.txt
```

## 1. Train the model

```bash
python train_model.py
```

This loads `data/train.csv`, cleans/encodes/engineers features, trains all
four classifiers on an 80/20 split, prints a comparison table, and saves the
best-performing model plus every preprocessing object the app needs into
`model/`.

Expected console output looks like:

```
Model: XGBoost
  Train Accuracy: 94.70%
  Test  Accuracy: 81.10%
...
BEST MODEL: XGBoost
```

> **Note:** exact accuracy numbers depend on your installed `xgboost`
> version and machine; if `xgboost` isn't installed, the script
> automatically falls back to `GradientBoostingClassifier` so the rest of
> the pipeline keeps working вЂ” install `xgboost` (`pip install xgboost`) to
> get the real XGBoost model.

## 2. Run the web app

```bash
python app.py
```

Open **http://127.0.0.1:5000** in a browser. Fill out the form and submit to
get an instant Approved / Declined read with a confidence score.

## Modeling notes

* **Missing values** вЂ” categorical fields imputed with the mode, numeric
  fields with the median/mode, matching the standard treatment for this
  dataset.
* **Feature engineering** вЂ” `TotalIncome` (applicant + co-applicant), plus
  log-transforms of `LoanAmount` and `TotalIncome` to tame their heavy right
  skew.
* **Encoding** вЂ” all categorical columns are label-encoded; everything is
  standardized with `StandardScaler` before being passed to any model
  (keeps KNN well-behaved; doesn't hurt the tree-based models).
* **Model selection** вЂ” whichever of the four classifiers scores highest on
  the held-out 20% test split is the one saved and served. Swap in a
  different selection rule (e.g. F1, AUC) in `select_best_model()` inside
  `train_model.py` if accuracy alone isn't the right metric for your use
  case вЂ” accuracy can be misleading here since the dataset's classes are
  imbalanced (~69% approved / 31% declined).

## Retraining on your own data

Replace `data/train.csv` with your own file using the same 13 columns
(`Loan_ID, Gender, Married, Dependents, Education, Self_Employed,
ApplicantIncome, CoapplicantIncome, LoanAmount, Loan_Amount_Term,
Credit_History, Property_Area, Loan_Status`), then re-run
`python train_model.py`. The Flask app will pick up the new model
automatically on its next restart.
