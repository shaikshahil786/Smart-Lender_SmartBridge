Pipeline

  1. Ingest Load applicant records: demographics, income, loan terms, credit history, and property area.
  2. Clean Impute missing categorical fields with the mode and missing numeric fields with the median. Normalize "3+" dependents bucket.
  3. Engineer Derive total household income, log-transform skewed fields (LoanAmount, TotalIncome) to tame outliers.
  4. Encode & scale Label-encode categorical fields, then standardize all features so every model sees comparable inputs.
  5. Train five candidates with CV Logistic Regression, Decision Tree, Random Forest, KNN, and XGBoost — each evaluated with 5-fold cross-validation, test accuracy, and ROC-AUC.
  6. Select & serve The candidate with the highest held-out test accuracy is persisted along with feature importances and loaded by this Flask app for real-time scoring.

Inputs used

Gender, marital status, dependents, education, employment status, applicant and co-applicant income, requested loan amount, loan term, credit history, and property area. None of these are stand-ins for protected-class screening — they mirror the fields lenders already collect on a standard application.
